# Анализ поведения пользователей интернет-магазина

## Цель проекта
Проект посвящен комплексному анализу поведения пользователей и покупок в интернет-магазине на основе трёх связанных датасетов: информация о пользователях, заказы и позиции товаров в заказах. Основная задача — выявление закономерностей покупательской активности, проблем с доставкой и сегментация клиентов для дальнейшего маркетинга и удержания.

## Исходные данные
- **df (пользователи)**: содержит уникальные идентификаторы клиентов, их географические данные (почтовый индекс, город, штат).  
- **df1 (заказы)**: включает информацию о заказах — идентификаторы клиентов и заказов, статус заказа, даты покупки, одобрения и доставки.  
- **df2 (товарные позиции)**: детализирует содержимое заказов — какие товары, по какой цене и каким продавцом были доставлены.  

## Методы и этапы анализа

### Анализ частоты покупок пользователей
Выявлены пользователи, совершившие только одну покупку. Это позволяет определить долю «одноразовых» клиентов и стратегию их удержания.

### Анализ причин недоставки заказов
Сгруппированы все заказы по статусам («shipped», «canceled», «unavailable», «invoiced» и др.) с расчетом среднего числа недоставленных заказов в месяц по каждой причине.  
Это помогает оценить проблемные точки в логистике и работу службы доставки.

### Анализ дня недели для покупок
Для каждого товара определен день недели, когда его покупают чаще всего.  
Это позволяет планировать акции и прогнозировать нагрузку на склад и службу доставки.

### Анализ частоты покупок по пользователям
Рассчитано среднее количество покупок на одного пользователя в неделю с учетом количества недель в каждом месяце.  
Этот показатель помогает оценить активность клиентов и сезонные колебания покупок.

### Когортный анализ пользователей
Построены когорты по месяцу первой покупки.  
Выявлены когорты с наибольшим retention на 3-й месяц, что позволяет оценить, какие группы пользователей наиболее ценны для бизнеса и лучше удерживаются.

### RFM-сегментация
Для каждого клиента рассчитаны метрики:
- **R (Recency)** — количество дней с момента последней покупки до текущей даты  
- **F (Frequency)** — суммарное количество покупок за всё время  
- **M (Monetary)** — сумма покупок за всё время  

На основе квантилей R, F, M выделены сегменты клиентов (например, RFM=132), для которых рассчитаны границы метрик: recency, frequency и monetary.  
Это позволяет определить лояльных клиентов, клиентов в оттоке, активных и потенциально ценных для маркетинговых акций.
